Buffers and Streams are core concepts in Node.js (not browser JavaScript) used for handling binary data and large data efficiently. They are extremely important for backend interviews and real-world systems like file uploads, video streaming, etc.

🔷 1. Buffers in JavaScript (Node.js)
👉 What is a Buffer?
A Buffer is a temporary storage for raw binary data.

JavaScript normally works with strings (UTF-16)

But systems like files, sockets, videos use binary data

Buffer helps bridge that gap

👉 Think of it like:

A fixed-size memory box where data is stored before processing

✅ Why Buffers?
Efficient handling of binary data

Used in:

File system (fs)

Network requests

Streams

Cryptography

🔹 Creating Buffers
// Create buffer of size 10 (empty)
const buf1 = Buffer.alloc(10);

// Create buffer from string
const buf2 = Buffer.from("Hello");

// Create buffer without initializing (faster)
const buf3 = Buffer.allocUnsafe(10);
🔹 Example: Buffer from String
const buffer = Buffer.from("Hello");

console.log(buffer); 
// <Buffer 48 65 6c 6c 6f>

console.log(buffer.toString()); 
// Hello
👉 Explanation:

Each character is stored as ASCII/UTF-8 bytes

Example:

H → 48

e → 65

🔹 Access & Modify Buffer
const buf = Buffer.from("ABC");

console.log(buf[0]); // 65 (ASCII of A)

// Modify
buf[0] = 97;

console.log(buf.toString()); // "aBC"
🔹 Important Methods
buffer.length
buffer.toString()
buffer.slice()
buffer.write()
🔥 Real-world Example (File Reading)
const fs = require("fs");

const data = fs.readFileSync("file.txt");
console.log(data); // Buffer output
console.log(data.toString()); // actual text
🔷 2. Streams in Node.js
👉 What is a Stream?
A Stream is a way to handle data piece by piece (chunks) instead of loading everything into memory.

👉 Think:

Instead of loading a whole movie, you watch it while downloading (Netflix style)

✅ Why Streams?
Without streams:

Large file → memory overload ❌

With streams:

Process in chunks → efficient ✅

🔹 Types of Streams
Node.js has 4 types:

Type	Description
Readable	Read data
Writable	Write data
Duplex	Read + Write
Transform	Modify data while streaming
🔹 Example: Reading File (Stream)
const fs = require("fs");

const readStream = fs.createReadStream("large.txt", {
  encoding: "utf-8",
});

readStream.on("data", (chunk) => {
  console.log("Chunk received:", chunk);
});

readStream.on("end", () => {
  console.log("Finished reading");
});
👉 Data comes in chunks, not all at once.

🔹 Example: Writing Stream
const fs = require("fs");

const writeStream = fs.createWriteStream("output.txt");

writeStream.write("Hello ");
writeStream.write("World");
writeStream.end();
🔹 Pipe (Most Important 🔥)
const fs = require("fs");

const readStream = fs.createReadStream("input.txt");
const writeStream = fs.createWriteStream("output.txt");

readStream.pipe(writeStream);
👉 This is like:

Connect source → destination directly

Memory usage	High	Low

Use case	Small data	Large files, videos

🔷 How Buffer + Stream Work Together
Streams use buffers internally

Each chunk = small buffer

👉 Flow:

File → Buffer (chunk) → Stream → Process → Output
🔥 Advanced Concept: Backpressure
👉 Problem:

Data coming faster than processing

👉 Solution:

Streams automatically control flow

Example:

Great question—this is exactly how interviewers test your understanding 👍
Let’s go through each type of stream with clear, practical examples.

🔷 1. Readable Stream (Read data)
👉 A stream you read data from

✅ Example: Reading a file
const fs = require("fs");

const readStream = fs.createReadStream("input.txt", {
  encoding: "utf-8"
});

readStream.on("data", (chunk) => {
  console.log("Chunk:", chunk);
});

readStream.on("end", () => {
  console.log("Finished reading");
});
🔍 What’s happening:
File is read in chunks

Each chunk is received via "data" event

🔷 2. Writable Stream (Write data)
👉 A stream you write data to

✅ Example: Writing to a file
const fs = require("fs");

const writeStream = fs.createWriteStream("output.txt");

writeStream.write("Hello ");
writeStream.write("World!");
writeStream.end();
🔍 What’s happening:
Data is written piece by piece

end() signals completion

🔷 3. Duplex Stream (Read + Write)
👉 A stream that can read and write simultaneously

✅ Example: TCP Socket (real-world)
const net = require("net");

const server = net.createServer((socket) => {
  socket.on("data", (data) => {
    console.log("Client says:", data.toString());

    // write back to client
    socket.write("Message received");
  });
});

server.listen(3000);
🔍 What’s happening:
socket is both:

Readable (receives data)

Writable (sends response)

🔷 4. Transform Stream (Modify data)
👉 A special duplex stream that modifies data while passing through

✅ Example: Convert text to uppercase
const { Transform } = require("stream");

const upperCaseTransform = new Transform({
  transform(chunk, encoding, callback) {
    const upper = chunk.toString().toUpperCase();
    callback(null, upper);
  }
});

// Use it
process.stdin
  .pipe(upperCaseTransform)
  .pipe(process.stdout);
🔍 What’s happening:
Input → transformed → output

Data is modified in real time

🔥 Real-world Transform Example (Compression)
const fs = require("fs");
const zlib = require("zlib");

fs.createReadStream("input.txt")
  .pipe(zlib.createGzip())
  .pipe(fs.createWriteStream("input.txt.gz"));
👉 Here:

zlib.createGzip() is a Transform stream

It compresses data while streaming

🧠 Quick Summary
Type	What it does	Example
Readable	Read data	fs.createReadStream()
Writable	Write data	fs.createWriteStream()
Duplex	Read + Write	net.Socket
Transform	Modify data	zlib, custom transform
